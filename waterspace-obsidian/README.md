# Wiki — второй мозг Фабрики

> Источник правды для долговременной памяти бренда. Живёт между сессиями. Индексируется через `MEMORY.md`.

Wiki — это **Obsidian-совместимый vault** в корне твоей установки Фабрики. Сюда копится всё, что должно пережить любую сессию: решения, фидбек, ссылки на ресурсы, состояния проектов, паттерны контента.

В отличие от `projects/{active}/memory/` (память одного проекта), wiki — это **сквозная память** по всем твоим проектам. Один бренд, два бренда, три ниши — wiki общий, проекты разные.

---

## Структура

**Типизированные заметки — плоско в корне с singular-префиксом:**

| Префикс | Тип | Что туда идёт |
|---|---|---|
| `feedback_*.md` | Правила, наученные на ошибках/успехах | «не делай X», «делай Y», тональные правила |
| `reference_*.md` | Указатели на внешние ресурсы | URL, ID, токены, схемы БД, доступы |
| `project_*.md` | Состояние/контекст проекта | стек, цели, активные инициативы, метрики |
| `decision_*.md` | Архитектурные решения | почему так, какие альтернативы отброшены |
| `user_*.md` | Профиль клиента | роль, экспертиза, стиль работы |

**Служебные папки:**

- `_setup/` — инструкции по установке qmd, Obsidian, конвенции
- `_templates/` — шаблоны frontmatter для новых заметок
- `inbox/` — входящие неразобранные заметки. Обрабатывает `wiki-keeper`.

**В корне:** `MEMORY.md` (индекс), `README.md` (этот файл).

---

## Почему плоско, а не папки

1. **Auto-memory совместимость.** Claude Code пишет в эту директорию плоско по convention `<type>_<topic>.md`. Папки = дубли в корне каждую сессию.
2. **Быстрый поиск в Obsidian.** `Cmd+O` по имени файла — мгновенно. Папки добавляют клики.
3. **Когерентный граф.** Все заметки на одном уровне → wikilinks работают без относительных путей → граф связей плотный.
4. **qmd-индекс.** Семантический поиск работает по всему vault без папочной фрагментации.

## Frontmatter

Все типизированные заметки имеют YAML frontmatter:

```yaml
---
name: Краткое имя на русском
description: Одна строка для индекса MEMORY.md
type: feedback | reference | project | decision | user
status: draft | active | stale | archived
audience: client | internal
updated: 2026-05-13
---
```

Шаблоны — в `_templates/`. Образцы — в `_setup/examples/`.

---

## MEMORY.md — главный hub

`MEMORY.md` подгружается автоматически в начале каждой сессии Claude (через `CLAUDE.md` в корне установки). Каждая заметка должна быть упомянута в `MEMORY.md` минимум одной ссылкой.

Формат строки:
```markdown
- [[feedback_topic|Заголовок]] — описание из frontmatter
```

`wiki-keeper` агент держит индекс актуальным и поднимает orphan-файлы в раздел `## Inbox`.

---

## Wikilinks обязательны

Для Obsidian-графа все связи между заметками — через `[[wikilinks]]`, не через markdown-ссылки.

```markdown
✅ См. [[decision_voice-rules]] и [[feedback_dash-killer]]
❌ См. [decision_voice-rules](decision_voice-rules.md)
```

Это работает в Obsidian (граф), в Claude Code (он понимает `[[...]]` как ссылку на файл), и в qmd (индексируется).

---

## Правила работы

- **Один файл — одна тема.** Если заметка стала смешанной — разбей на две через ссылки.
- **Не создавай папки `feedback/`, `projects/`, `decisions/`** — только префиксы в корне.
- **Не удаляй заметки** — помечай `status: archived` или переноси в архивную секцию `MEMORY.md`.
- **Backup перед массовыми изменениями.** Wiki — это многомесячный труд, не теряй его.
- **Каждая заметка в `MEMORY.md`.** Без ссылки в индексе заметка теряется.

## Установка

1. **qmd MCP** (рекомендуется) — семантический поиск через embeddings. См. [[_setup/qmd-install]].
2. **Obsidian** (рекомендуется) — визуальный редактор + граф связей. См. [[_setup/obsidian-setup]].
3. **Конвенции** — детальные правила имён и frontmatter. См. [[_setup/conventions]].

Wiki работает и без этих инструментов: любой текстовый редактор + поиск по grep. Но с ними — в десять раз быстрее.

---

## Что НЕ хранить в wiki

- **Контент** (посты, статьи, скрипты) → `projects/{active}/outputs/{module}/`
- **Активная задача** → `projects/{active}/memory/active-context.md`
- **Стилевые правки одного проекта** → `projects/{active}/memory/feedback.md`
- **Секреты и токены** → не сюда. Используй переменные окружения или менеджер паролей.

Wiki — для знаний, которые переживают любой проект.
